Continuous Documentation

Continuous Delivery und Continuous Deployment (CI/CD) sind grundlegende Prozesse von Softwareentwicklung mit DevOps. Geht das auch mit Dokumentation?

User Story: Wir bauen einen Blog. Wie? Mit Hilfe von CI/CD und einem Static Website Generator.

Fünf Vorraussetzungen

1) Eigener Laptop mit Recht, Software zu installieren,
2) WLAN Zugang zu ‘gov-public’ ist eingerichtet,
3) funktionierender Login bei GitHub.com,
4) installierte IDE oder Texteditor, z.B. https://code.visualstudio.com/
5) installiertes Git https://git-scm.com/downloads

Los geht's

Fünf Schritte zu deinem Blog

  1. Account eröffnen auf netlify.com
  2. Installieren Academic: https://sourcethemes.com/academic/ mit diesem Link auf Netlify
  3. Öffnen des Blogs in Github.com
  4. In Github Editor die Seite 
    /content/author/admin/_index.md
    ändern und speichern.
  5. Änderung im Blog anschauen.
  6. Optional: Git Repo clonen und lokal editieren, committen, etc.
